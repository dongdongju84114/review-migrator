# 경영지원팀용 리뷰 이전 도구 사용법

## 언제 쓰나요?

네이버 스마트스토어 리뷰 엑셀을 크리마 리뷰 등록 파일로 바꿀 때 사용합니다.

## 실행 방법

macOS에서는 `run_review_migrator_gui.command` 파일을 더블클릭합니다.

Windows에서 `ReviewMigratorGUI.exe`를 받은 경우에는 그 파일을 더블클릭합니다. 이 방식은 Python 설치가 필요 없습니다.

EXE가 없고 `run_review_migrator_gui.bat` 파일을 받은 경우에는 `.bat` 파일을 더블클릭합니다. Python 3.11 이상이 없으면 설치 안내가 나오며, PC에서 `winget`을 사용할 수 있으면 Python 자동 설치를 선택할 수 있습니다. 첫 실행 때 필요한 패키지를 설치하므로 인터넷 연결이 필요할 수 있습니다.

## 화면에서 선택할 것

1. `네이버 리뷰 엑셀`: 스마트스토어에서 다운로드한 리뷰 엑셀 파일
2. `마켓플러스 CSV`: 카페24 마켓플러스에서 네이버 스마트스토어로 필터해서 다운로드한 상품 CSV입니다.
3. `카페24 상품 CSV`: 카페24 상품 전체 목록 CSV입니다. `상품코드`와 `상품번호` 컬럼이 있어야 합니다.

결과는 자동으로 기본 결과 폴더에 저장됩니다. Windows EXE는 `내 문서\ReviewMigrator\operator_runs`, macOS/개발 폴더 실행은 실행 폴더의 `operator_runs`를 사용합니다. `.env`와 이미지 공개 URL은 미리 준비된 기본 설정을 사용합니다.

리뷰 이미지는 네이버 엑셀의 `포토/영상` URL에서 자동으로 내려받고, 실제 등록 때 Cafe24 FTP로 올라갑니다.

## 안전 검증 순서

1. `안전 검증 파일 만들기` 버튼을 누릅니다.
2. 실행이 끝나면 `결과 폴더 열기`를 누릅니다.
3. `run_summary.html`을 엽니다.
4. 상태가 `검토 필요`이면 아래 파일을 확인해서 수정합니다.
   - `failed_mapping.csv`
   - `product_mapping_review_required.csv`
   - `image_matches_review_required.csv`
   - `downloaded_image_manifest.csv`
   - `normalize_issues.jsonl`
   - `payload_issues.jsonl`
5. 상태가 `업로드 가능`이면 실제 등록 단계로 넘어갈 수 있습니다.

## 크리마 권한 확인

실제 등록 전에 `크리마 권한 확인` 버튼을 누릅니다.

- `크리마 리뷰 조회 권한`이 실패하면 크리마 API 인증부터 확인해야 합니다.
- `크리마 상품 조회 권한`이 실패하면 크리마 관리자/API 앱에 `Product API 상품 조회 권한`을 추가해야 합니다.
- `크리마 리뷰 생성/수정 권한`은 실제 리뷰를 만들지 않고는 완전히 확인할 수 없습니다. 담당자가 크리마 앱 권한에서 확인해야 합니다.

크리마 담당자에게 요청할 권한:

- Product API 상품 조회 권한: `GET /v1/products`
- Review API 리뷰 조회 권한: `GET /v1/reviews`
- Review API 리뷰 생성 권한: `POST /v1/reviews`
- Review API 리뷰 수정 권한: `PATCH /v1/reviews`

권한을 추가한 뒤에는 새 토큰을 발급하거나 `.env`의 토큰 정보를 갱신한 다음 `크리마 권한 확인`을 다시 실행합니다. 실행 중 토큰이 만료되면 도구가 새 토큰을 받아 `.env`에 자동 저장합니다.

## 실제 크리마 등록

실제 등록은 담당자 확인 후에만 진행합니다.

1. `안전 검증 파일 만들기`를 먼저 실행합니다.
2. `run_summary.html` 상태가 `업로드 가능`인지 확인합니다.
3. `크리마 권한 확인`을 실행하고 필수 권한이 통과했는지 확인합니다.
4. 화면에서 `실제 등록 승인`을 체크합니다.
5. `실제 크리마 등록 실행` 버튼을 누릅니다.
6. 이때만 이미지가 Cafe24 FTP로 업로드되고, 이후 크리마에 리뷰가 등록됩니다.
7. 등록 후 결과 폴더의 `verification_report.html`을 확인합니다.

## 주의

- `안전 검증 파일 만들기`는 크리마에 등록하지 않습니다.
- `안전 검증 파일 만들기`는 Cafe24 FTP에도 이미지를 올리지 않습니다. 결과 파일에는 업로드 예정 URL만 표시됩니다.
- `실제 크리마 등록 실행`은 크리마에 리뷰를 생성하거나 수정할 수 있습니다.
- `failed_mapping.csv`에 내용이 있으면 상품번호 연결부터 수정해야 합니다.
- `product_mapping_review_required.csv`에 내용이 있으면 자동 매칭이 확실하지 않은 상품입니다.
- `마켓플러스 CSV`의 `P000...` 상품코드는 카페24 상품코드입니다. `카페24 상품 CSV`를 같이 넣으면 도구가 카페24 `상품번호`로 바꿔 크리마 상품과 맞춥니다.
- `downloaded_images` 폴더에는 네이버에서 다운로드한 이미지가 저장됩니다.
- 네이버 이미지 URL은 크리마에 그대로 넣지 않습니다. 공개 저장소에 올린 뒤 그 공개 URL만 크리마 CSV/API에 들어갑니다.
- 이미지가 크리마에 보이지 않으면 결과 폴더의 `image_public_url_checks.csv`와 `.env`의 `CAFE24_IMAGE_BASE_URL`을 담당자에게 확인 요청해주세요.
